What's new

Version 0.4

Release date: 7 April, 2008

Version description

Seemes 0.4 is an update to Seemes 0.4 Beta 1, including bug fixes and feature updates, and is also the first 0.4 release marked as "stable."

New features

  • A control panel has finally been added for managing files in the image directory.
  • An easy way to add links to other pages within Seemes has been added to the Create page and Edit page control panels. (Note: This feature is non-compatible with IE6 and previous.)
  • Increased the maximum upload size in the File manager to 100Mb. (Note: If you still can't upload large files, check your php.ini's value for upload_max_filesize and post_max_size.)
  • Minor tweaks to the default theme.
  • The HTML used for page headers can now be changed via a variable in config.php.
  • The menu customization now lets you specify extra menu item HTML when that item is for the current page.
  • The year on the default copyright notice in the installer has been updated from 2007 to 2008.

Bug fixes

  • Quotation marks in the website title and copyright are now correctly saved by the installer.

Version 0.4 Beta 1

Release date: 7 December, 2007

Version description

Seemes 0.4 Beta 1 is an update to Seemes 0.3, including bug fixes and feature updates.

New features

  • The installer has been rewritten. New features include:
    • Cleaner, 3-step installation process
    • Permissions check before installation
  • The menu system has been rewritten to allow customization of menu output, as well as easier adding of other menu features down the road.
  • The old theme has been replaced.
  • You can now set themes for individual pages, as well as globally.

Bug fixes

  • Fixed incorrect handling of HTML entities when loading the extra content for page editing.
  • When editing pages, /'s inside HTML comments no longer become &#x2F;'s.

Things that'll break

  • Because of modifications to the way extra files (the files that store extra page-specific data, such as extra <head> data) have been reworked to allow for page-specific themes, old extra files will be incompatible with Seemes 0.4. Please see upgrade.htm for upgrade instructions. To upgrade old extra files:
    1. Download an extra file from your data folder to your computer.
    2. Open it in your favourite text editor. Notepad is probably not a good choice in this instance, because it won't handle line breaks properly if you've been using a *nix-based server.
    3. Your file should have a format something like this: 
       onload="javascript:myFunction('foo', 42);"
<script type="text/javascript" />
function myFunction(myString, myInt) {
	alert(myString + myInt);
}
</script>
      The first line indicates extra content inside the actual <body> tag. Everything after that is extra <head> content.
    4. Insert a blank line between line 1 and line 2, so your file looks like this: 
       onload="javascript:myFunction('foo', 42);"

<script type="text/javascript" />
function myFunction(myString, myInt) {
	alert(myString + myInt);
}
</script>
      The new line allows room to save data about what theme the page should use.
    5. Save the file and upload it.
    6. Now, repeat the process for all extra files.

Version 0.3

Release date: 7 August, 2007

Version description

Seemes 0.3 is a full-version upgrade from Seemes 0.2, including bug fixes and feature updates, and is also the first 0.3 release marked as "stable."

New features

  • All the file management features have been moved into a single control panel called "File manager."
  • Normal menu items can now have bulleted submenu items that appear only when the user is on that page.
  • Theme files can now be edited from within the Admin Control Panel.
  • The built-in functions listpages() and listfiles() now take an argument indicating what page or file should be removed from the list (multiple items can be passed as an array).

Bug fixes

  • &'s are now automatically converted to &amp; when loading the page for editing.
  • All textareas in the control panels for the main content of things are now 80 columns by 25 rows to reflect the standard console size, rather than the 60 columns by 15 rows of before (which didn't reflect anything). (Note that if you're using a screen resolution that won't fit 80x25, the default theme sets the maximum width for textareas to 100% so this should cause no width problems.)
  • Blank lines in the menu file no longer cause an error.
  • The built-in function listfiles() now returns all files in the website upload directory and not just those with extensions.
  • The default theme now validates fully.
  • When a file is uploaded, Seemes now CHMODs it 0666.
  • When a page is created, Seemes now CHMODs it 0666.

Version 0.2

Release date: 7 July, 2007

Version description

Seemes 0.2 is an update to Seemes 0.2 Beta 2, including bug fixes and feature updates, and is also the first 0.2 release marked as "stable."

New features

  • If config.php is not found, Seemes now exits with a nice error instead of just spitting out the PHP error.

Bug fixes

  • Quotation marks and appostrophies are now properly handled in the website title and copyright line. Special characters are no longer converted to HTML entities automatically; if an HTML entity is wished for, it must be entered manually.

Version 0.2 Beta 2

Release date: 7 June, 2007

Version description

Seemes 0.2 Beta 2 is an update to Seemes 0.2 Beta 1, including bug fixes and feature updates.

New features

  • External links may now be placed in the main menu.
  • Filename entry is now done using a dropdown box instead of a text input box.
  • Menu items now have their own ID's.
  • There is now a page load timer; access the current load time by calling timerreturn().
  • You can now specify how a page should be added to the menu when creating the page.

Bug fixes

  • </textarea>'s no longer terminate the editing space in the Edit page control panel. (This is done by converting all instances of / to the element &#x2F; when the page is loaded for editing; the symbol is automatically reverted to / when the page is saved.)
  • Slashes are now stripped from page names upon page load.
  • The name of the temp variable used to store the content of the menu while loading the Edit menu control panel has been changed from $menu to $menucontent to avoid overwriting the formatted menu. (This bug made menu editing difficult while using theme files that placed the menu after the page content in the page's HTML.)
